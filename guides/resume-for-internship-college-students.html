Resume for Internship: Format for College Students

Applying for internships with an empty "experience" section feels hopeless — but companies offering internships expect zero experience. What they're actually judging: can you show initiative through projects, coursework and skills. Here's the format that does it.

Section order for an internship resume

  1. Header — name, phone, email, city, LinkedIn/GitHub if you have them
  2. Summary — 2 lines: year + branch + what internship you want + one proof point
  3. Projects — your experience substitute; 2–4 entries
  4. Education — degree, college, expected year, CGPA if 7.5+
  5. Skills — tools, languages, software you can actually use
  6. Extras — hackathons, clubs, certificates, sports leadership

Sample summary

Third-year B.Tech (IT) student seeking a summer software internship. Built a hostel-mess feedback app used by 200+ students; comfortable with Python, Git and SQL.

How to write project entries (the part that matters)

Each project gets a title, one line on what it does, one line on your role/result:

Hostel Mess Feedback App — web app where students rate daily meals.
Built the backend in Flask; 200+ active users; mess committee adopted it officially.
Sales Dashboard (coursework) — analysed a 10k-row retail dataset in Excel/Power BI; presented insights that identified the top 3 loss-making categories.
✅ Numbers make projects real: users, rows of data, marks, downloads, team size. Even small numbers beat none.

No projects yet?

Make one this week — genuinely. A small personal project (expense tracker, class-notes site, Excel analysis of public data) started today is a resume line by Friday, and interviewers love "I built this last month" stories.
